## 问题根源

### 1. 搜索功能失效的根本原因

**Stack 主题的设计缺陷**：
- 主题的 `layouts/page/search.html` 使用 `JSXFactory: "createElement"` 编译 JSX
- 但没有在运行时提供 `createElement` 函数的定义
- 导致 `ReferenceError: createElement is not defined`

**原始代码**（有问题的版本）：
```html
{{- $opts := dict "minify" hugo.IsProduction "JSXFactory" "createElement" -}}
{{- $searchScript := resources.Get "ts/search.tsx" | js.Build $opts -}}
<script type="text/javascript" src="{{ $searchScript.RelPermalink }}" defer></script>
```

缺少 `window.createElement` 函数定义。

## 修复方案

### 1. 搜索功能修复

**文件**：`theme-profiles/study/layouts/page/search.html`

**添加内容**：
```html
<script>
    window.searchResultTitleTemplate = "{{ i18n `search.resultTitle` }}"

    // Simple createElement function for JSX
    window.createElement = function(tag, props, ...children) {
        const element = document.createElement(tag);

        if (props) {
            if (props.className) element.className = props.className;
            if (props.class) element.className = props.class;
            if (props.href) element.href = props.href;
            if (props.src) element.src = props.src;
            if (props.loading) element.loading = props.loading;
            if (props.dangerouslySetInnerHTML && props.dangerouslySetInnerHTML.__html) {
                element.innerHTML = props.dangerouslySetInnerHTML.__html;
            }
            for (const key in props) {
                if (!['className', 'class', 'href', 'src', 'loading', 'dangerouslySetInnerHTML'].includes(key)) {
                    element.setAttribute(key, props[key]);
                }
            }
        }

        children.forEach(child => {
            if (child == null) return;
            if (typeof child === 'string' || typeof child === 'number') {
                element.appendChild(document.createTextNode(child));
            } else if (child instanceof Element) {
                element.appendChild(child);
            } else if (Array.isArray(child)) {
                child.forEach(c => {
                    if (c == null) return;
                    if (typeof c === 'string' || typeof c === 'number') {
                        element.appendChild(document.createTextNode(c));
                    } else if (c instanceof Element) {
                        element.appendChild(c);
                    }
                });
            }
        });

        return element;
    };
</script>
```

### 2. i18n 文件修复

**错误**：`i18n/zh.toml`
**正确**：`i18n/zh-cn.toml`

必须与 `hugo.toml` 中的 `defaultContentLanguage = "zh-cn"` 匹配。

### 3. 完整的修复文件清单

```
theme-profiles/study/
├── layouts/
│   ├── home.html          ← 添加 i18n 调用
│   ├── archives.html       ← 添加 i18n 调用和中文日期
│   └── page/
│       └── search.html     ← 添加 createElement 函数
└── config.toml            ← 确认 defaultContentLanguage

i18n/
└── zh-cn.toml             ← 添加所有翻译键
```
